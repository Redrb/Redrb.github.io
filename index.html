¡Hola mundo!

Mi nombre es Oscar y soy de Cajamarca. Me gradué en Ingeniería de Sistemas en la Universidad Nacional de Cajamarca.

Mi enfoque abarca tanto el backend como el frontend, disfruto de programar, maquetar y refactorizar(en caso de ser necesario), muchas veces me encuentro tan inmerso que pierdo la noción del tiempo.

Esta página web es mi manera de expresarme y compartir todo lo que me apasiona, especialmente en el campo de la programación, desarrollo web, desarrollo móvil y todas las curiosidades que considero relevantes. My knowledge

C#

Es el primer lenguaje aprendí y apliqué en entornos profesionales, hablar de este lenguaje me trae muy buenos recuerdos, cuando lo use fue para el backend de un proyecto de monitoreo de SSOMA, aquí aplique conocimientos de LINQ, inyeccion de dependecias, mis primeras nociones sobre migraciones, creación de API seguras(utilizando jwt) además de un poco de frontend con razor.

PHP

Debo admitir que al inicio no era de mis favoritos, sin embargo cuando se usa de manera ordenada, aplicando clean code y de la mano de algún framework como laravel se convierte en uno de mis preferidos, con php hice algunas integraciones con gmail y outlook.

Javascript

Cuando lo vi por primera vez me pareció un poco confuso ya que no es fuertemente tipado, en aquel momento incluso no se usaba las definiciones let y const, sino que todo se realizaba con definiciones del tipo var; Adicionalmente se agregaba undersore y jquery(actualmente ya son legacy) lo cual como novato en aquel momento me confundió aún más, sin embargo en la actualidad ya no es un problema, la animación que ves al lado izquierdo la hice usando js en pocos minutos.

Dart

A mi parecer es un mezcla de c# con js, inicié a usarlo para poder realizar aplicaciones móviles y es bastante entretenido de aprender, me encanta la forma de usar las promesas muy similar a js, esto se acopla muy bien cuando se integra con Flutter debido a los builders que proporciona.

Vue js

Fue el segundo framework js que vi y el que más usé; Usé options api y en el futuro pienso practicar composition api, de igual manera que con angular, la reactividad de los modelos me parece impresionante, nunca me voy a dejar de asombrar de esos aspectos, supongo que cuando vea react me pasará lo mismo. Desarrollé un editor html para personas que no tengan nociones de html, esto con el fin de maquetar documentos según las preferencias del usuario, no puedo mostrar capturas, ya que fue para una empresa privada y considero que lo que haya desarrollado para mis anteriores trabajos es reservado.

Angular js

Es el primer framework js que usé, aquí obtuve nociones de los modelos reactivos, ver como cambiaba algo en tiempo real en la interfaz de usuario me pareció algo increíble y útil, usar los loops facilitaba el trabajo y los watcher, para estar vigilando siempre si algún modelo cambiaba y agregar eventos en función al cambio que se esté realizando, mostrar alertas.

Node js

Debido a que usa js me pareció bastante sencillo, mi principal uso de node js fue para realizar web scraping con puppeter, como recomendación personal y debido a que se usan mayormente promesas, trata de hacer que tus funciones sean asíncronas para usar async / await ya que si usas .then, para los casos de scrapping se van a anidar demasiado (por los tiempos de espera de elementos, navegaciones, etc.) y sería un poco problemático revisar el código, también puedes usar las anidaciones en el .then pero asegurate de sacar el contenido a una función para que mantengas el orden.

SQL server

Hice mis primeras migraciones, vistas y consultas simples aquí la estudié en la universidad y tuve la oportunidad de aplicar los conocimientos mientras realizaba mis prácticas profesionales; Ya que el ecosistema en el que se encuentra es de microsfot facilitó el desarrollo de aplicaciones por su fácil integración cuando lo use con .net framework.

Asp.net

Debido a que va de la mano con c#, lo vi por primera vez en la universidad y profundicé estos conocimientos en un entorno profesional, la forma en la que se va estructurando el proyecto me parece increíble debido al orden generado por usar MVC, lo que facilita la separación de la lógica de presentación, la lógica de negocios y la manipulación de datos. Esto resulta en un código más modular, mantenible y escalable. Así mismo si existe posibilidad de usar librerías de clases para mantener un mejor orden me parece un acierto tremendo, luego de ello únicamente se tendría que agregar las referencias en los lugares que las necesiten.

MySQL

Es el que más he usado debido a que es gratis, en primera instancia creaba mis bases de datos y tablas con línea de comandos sin embargo para facilitar las cosas empecé a usar la herramienta visual MySQL Workbench, admito que tuve algunos problemas de compatibilidad de versiones en algunas ocaciones a la hora de conectar las base de datos de con algún framework, sin embargo todos los he resuelto satisfactoriamente.

Laravel

La primera versión que vi fue laravel 5, poco a poco me fui familiarizando con sus características, con eloquent y los comandos de php artisan para generar migraciones, controladores y modelos de manera mucho más veloz, luego hice upgrade a laravel 8, aquí fue cuando fui agregando mas cosas del mismo ecosistema de laravel como laravel sanctum y larvel dusk además terminé integrando vue js debido a que es escalable y solo se usaba en cierta sección del proyecto ya que se requería de la reacitivad de modelos que vue proporciona.

Flutter

Me decidí por usar flutter, ya que cuando se compila el resultado final puede obtenerse una archivo instalable par android y ios, la forma en que se estructuran los widgets con filas, columnas, contenedores, etc. me parecía bastante parecido al sistema de grillas de bootstrap por lo cual me fue sencillo realizar las maquetaciones de las interfaces de usuario, algo que me parece muy interesante los builder que posee, streambuilder y future builder atrabajando con stream y futures(promesas) según corresponde, Flutter te da total libertad de estructurar tus carpetas como desees pero considero estructurar las carpetas en orden principalmente agrupando tus modelos, repositorios, datasources, mappers, vistas y widgets mínimamente. He usado riverpod como gestor de estado, usaré más según sea necesario para el proyecto que esté colaborando en ese momento.
